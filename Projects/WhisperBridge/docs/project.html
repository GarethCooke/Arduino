WhisperBridge

BLE → MQTT bridge for the Vent-Axia Svara bathroom fan. Sits on your WiFi network and translates Home Assistant commands into Bluetooth sequences.

Overview

WhisperBridge is an ESP32-based bridge that allows a Vent-Axia Svara bathroom extractor fan to be controlled by Home Assistant (or any MQTT client). The fan itself is only controllable via Bluetooth Low Energy — there is no WiFi on the fan. WhisperBridge provides the translation layer: it sits on your WiFi network, subscribes to an MQTT topic, and when an ON command arrives it connects to the fan over BLE, authenticates, sends the boost command, and disconnects.

Because the ESP32 handles all BLE communication wirelessly, there is no physical wiring to the fan. The only real-world connections are power to the ESP32 and optionally a status LED.

BLE Sequence

WhisperBridge performs this sequence on every boost command:

1
Connect to the fan by MAC address
2
Write 4-byte PIN to auth characteristic: e3 14 62 05
3
Wait 200 ms for the fan to validate the PIN
4
Write 5-byte boost payload to command characteristic: 01 60 09 84 03
5
Disconnect cleanly
info
The entire BLE sequence runs in a dedicated FreeRTOS task so the WiFi stack, MQTT client, and web server continue to respond while the (slow) BLE connection is in progress.

The Vent-Axia Svara communicates entirely over BLE — no wires connect the ESP32 to the fan. Place the ESP32 within BLE range. The firmware sets TX power to maximum (ESP_PWR_LVL_P9, +9 dBm) to maximise range through walls.

Power

The ESP32 DevKit V1 can be powered in two ways:

  • Micro-USB — plug in any 5 V USB charger (1 A or greater recommended). Also used for first-time flashing.
  • VIN pin — connect 5 V to the VIN pin and ground to any GND pin for hardwired installations.
warning
Do not connect 3.3 V to the VIN pin. It bypasses the regulator and will under-power the ESP32. The board requires 5 V on VIN (or via USB). The ESP32 chip itself runs at 3.3 V internally.

Optional Status LED

The DevKit V1 has a built-in blue LED on GPIO2 — no external component needed. For a brighter external indicator, wire GPIO2 → 330 Ω resistor → LED anode → LED cathode → GND.

Software Setup

1

Configure credentials

Edit WhisperServer/include/config.h before flashing:

config.hcontent_copy
// MQTT broker
#define MQTT_HOST     "192.168.1.xxx"   // your broker IP
#define MQTT_USER     "user"            // leave "" if no auth
#define MQTT_PASSWORD "password"

// Fan MAC address (find with a BLE scanner app)
#define FAN_MAC_ADDRESS "AA:BB:CC:DD:EE:FF"

// OTA password
#define OTA_PASSWORD "choose-something-secure"
2

Flash via USB (first time)

terminal
cd WhisperServer
pio run -e esp32dev -t upload
3

WiFi setup

On first boot with no saved credentials the device starts in AP mode. Connect to the WhisperBridge-Setup WiFi network (password: whisperbridge), then browse to http://10.0.0.1/setup.html. Select your network, enter the password, and tap Connect. The device restarts and joins your network.

4

Home Assistant MQTT discovery

Once connected, the device publishes an MQTT auto-discovery payload automatically. Home Assistant will create a switch entity named Boost under a WhisperBridge device. Toggling it ON triggers the fan boost sequence.

hub

MQTT Topics

Topics are unique per device using the last 3 bytes of the MAC address (e.g. a1b2c3):

TopicDirectionPayload
whisperbridge/<id>/boost Subscribe (command) ON or 1 triggers boost; OFF is ignored (boost is one-shot)
whisperbridge/<id>/boost/state Publish (state, retained) ON when BLE sequence starts; OFF when it completes
homeassistant/switch/whisperbridge_<id>_boost/config Publish (retained) Home Assistant auto-discovery JSON
